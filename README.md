# PirateSwap closed bot — деплой и эксплуатация

Закрытый Telegram-бот для фиксированного списка из 3 человек. Мониторит появление
кейсов на PirateSwap и шлёт алерты. Мониторинг и рассылка разделены: опрос сайта
идёт всегда, `/stop` отключает уведомления только нажавшему.

## Состав репозитория

| Файл | Назначение |
|------|-----------|
| `pirateswap_bot.py` | весь бот (поллер + Telegram + HTTP /health) |
| `requirements.txt` | пустой (stdlib-only), нужен Render для распознавания Python |
| `render.yaml` | Blueprint для Render (опционально) |
| `.gitignore` | чтобы не закоммитить state/логи/`.env` |

`notify_state.json` и `events.jsonl` создаются в рантайме — **не коммить их**.

---

## ⚠️ Шаг 0. Отозвать засвеченный токен

Старый токен был захардкожен в исходнике и засветился. Открой **@BotFather →
`/revoke`** для этого бота, получи **новый** токен. В коде токена больше нет —
он читается только из переменной окружения.

---

## Шаг 1. Активация чатов (один раз)

Telegram не даёт боту писать первым. Поэтому **каждый из троих** должен один раз
открыть бота и нажать **/start**. Это не «регистрация новых людей» — белый список
жёстко зашит в коде (`682446275, 770511678, 778333031`), `/start` лишь
разблокирует канал «бот → пользователь». Любой другой chat_id игнорируется молча.

## Шаг 3. Создать Web Service на Render

**Важно:** именно **Web Service**, не Background Worker — на free-плане воркер не
живёт 24/7, а web-сервис с пингами на `/health` не засыпает.

Вручную:
1. Render → **New + → Web Service** → подключить репозиторий.
2. Настройки:
   - **Runtime:** Python 3
   - **Build Command:** `pip install -r requirements.txt`
   - **Start Command:** `python pirateswap_bot.py`
   - **Instance Type:** Free
3. **Environment → Add Environment Variable:**
   - Key: `PIRATE_TG_TOKEN`, Value: *новый токен от BotFather*
   - (порт `PORT` Render проставляет сам — бот его читает.)
4. **Create Web Service.**

Либо через `render.yaml`: **New + → Blueprint**, токен ввести в дашборде
(в blueprint он `sync: false` — в код не попадает).

---

## Шаг 4. UptimeRobot — чтобы Render не засыпал

Free Web Service засыпает после ~15 мин без HTTP-трафика. Пинг не даёт уснуть:

1. На Render скопируй URL сервиса, напр. `https://pirateswap-bot.onrender.com`.
2. UptimeRobot → **Add New Monitor**:
   - Type: **HTTP(s)**
   - URL: `https://<твой-сервис>.onrender.com/health`
   - Interval: **5 минут**
3. Save. `/health` отвечает `200 OK`.

## Что смотреть в логах Render (Logs)

- `HTTP-сервер слушает 0.0.0.0:<port>` — web-часть поднялась.
- `Поток-поллер запущен` + `Тихий стартовый скан` — мониторинг пошёл.
- `[Prisma Case] baseline: N шт.` где **N > 0** — бот реально видит сток
  (Prisma всегда в наличии). Если 0 — фильтр/сеть сломаны.
- `🔥 ДРОП` — сработал алерт.
- `429` / `HTTP 403/503 (Cloudflare?)` — притормаживаем (backoff отрабатывает сам).
- `⚠️ отброшено чужих предметов` — анти-ложная защита поймала сбой фильтра
  (алерт по ним НЕ ушёл).

Проверить, что бот жив, можно в любой момент: написать ему **/status**.

## Поведение, о котором стоит помнить

- **Эфемерная ФС Render.** При каждом рестарте/редеплое `notify_state.json`
  обнуляется → все трое снова `notify=true`. Кто хочет тишины — жмёт 🔕 заново.
  Для троих это сознательный компромисс (так и решили). `events.jsonl` тоже
  обнуляется при рестарте — если нужна история расписания надолго, периодически
  скачивай его из Render Shell.
- **Тихий старт.** На старте бот фиксирует baseline без алертов — иначе каждый
  редеплой слал бы «уже в наличии». Алерты идут только на дропы ПОСЛЕ запуска.
- **Round-robin опрос (для скорости покупки).** Боевые предметы опрашиваются по
  кругу по одному с паузой 4–6 с, без длинных простоев → каждый перепроверяется
  ~раз в 25 с (быстрее детект = больше шанс успеть купить). Контрольный Prisma
  опрашивается редко (`CONTROL_EVERY_SEC`, по умолч. 600 с). Алерт короткий,
  с кнопкой **🛒 Купить** (тап открывает предмет).
- **Cloudflare с серверного IP.** Человеческий User-Agent + backoff на 429/5xx.
  Средний темп ~1 запрос / 5 с. Если Render начнёт ловить `1015` — увеличь
  `ITEM_GAP_MIN/MAX` в начале `pirateswap_bot.py`.
- **Heartbeat** (`❤️ Бот жив`, раз в 6 ч) шлётся только админу (`770511678`).
- **Админ-алерт на серверные ошибки.** При HTTP 5xx или Cloudflare 403/503 админу
  (`770511678`) приходит ЛС `🛑 Серверная ошибка PirateSwap`. Чтобы при череде
  ошибок не было спама — не чаще одного сообщения раз в 15 мин
  (`ERROR_ALERT_COOLDOWN_MIN`).

## 🛒 Автопокупка (опционально, выключена по умолчанию)

Бот может сам оформлять покупку в момент детекта (`POST /Exchange/start/v2/{SteamID}`),
быстрее, чем человек успеет тапнуть кнопку.

**Закупки — только для админа.** Покупка идёт на одном аккаунте (чей `PIRATE_BEARER`
и `PIRATE_STEAMID` заданы) — это админ (`770511678`). Поэтому:
- **Двое других** (`682446275`, `778333031`) получают **только уведомления** о
  дропе + кнопку 🛒 «Купить» для ручной покупки. Результатов закупки они не видят.
- **Админ** получает результат закупки (`✅ Куплено` / `🧪 dry-run` / `⚠️ не успел`
  / `🔑 токен протух`) **отдельным сообщением и всегда** — даже если приглушил
  уведомления кнопкой 🔕. То есть админ может нажать 🔕 Стоп и получать **только
  закупки**, без обычных дроп-уведомлений.

> ⚠️ **Риски.** Тратит **реальные деньги** автоматически и почти наверняка
> **против ToS PirateSwap** — возможен **бан аккаунта**. Включаешь на свой риск.

### Двойной предохранитель

1. **`AUTOBUY_ENABLED` (env)** — мастер-переключатель фичи.
2. **Цены на кейсы задаёшь через `/menu`** — даже при `AUTOBUY_ENABLED=1` бот
   НИЧЕГО не покупает, пока админ не выставит цену на кейс. Цены живут **в памяти
   и сбрасываются при рестарте** — после каждого перезапуска Render цены пустые,
   бот только уведомляет, пока ты их заново не выставишь через `/menu`.

### Как задать цены на кейсы

1. В env `AUTOBUY_ITEMS` перечисли кейсы через запятую — **точно как в `TARGETS`**
   (иначе не совпадёт и покупаться не будет; бот залогирует такие имена).
   Пример: `AUTOBUY_ITEMS=Revolution Case,Fracture Case,Kilowatt Case`
2. В Telegram отправь **`/menu`** → появятся кнопки по кейсам.
3. Нажми кейс → бот попросит цену → пришли число (напр. `0.31`).
4. Готово: бот купит этот кейс, когда он дропнется по цене ≤ заданной.
   Повтори для каждого кейса. Кнопка «🔴 Сбросить все цены» обнуляет всё.

### Переменные окружения (Render → Environment)

| Переменная | По умолч. | Смысл |
|---|---|---|
| `AUTOBUY_ENABLED` | `0` | `1` — включить фичу автозакупки (иначе только алерты) |
| `AUTOBUY_DRY_RUN` | `1` | `1` — только симуляция (ничего не тратит); `0` — боевой режим |
| `AUTOBUY_ITEMS` | `Revolution Case` | какие кейсы разрешено покупать (через запятую, имена как в `TARGETS`) |
| `AUTOBUY_DAILY_LIMIT` | `20` | макс. покупок в сутки (счётчик в памяти, сброс при рестарте) |
| `AUTOBUY_PRICE_FIELD` | `price` | какое поле цены слать: `price` или `storePrice` |
| `AUTOBUY_MAX_PRICE` | `0` | *необязательный* жёсткий потолок для вводимой цены (защита от опечатки). `0` = без потолка |
| `PIRATE_BEARER` | — | **Bearer-токен сайта** (из запроса `start/v2`). Протухает! |
| `PIRATE_STEAMID` | — | твой SteamID64 (публичный, из URL запроса) |

### Команды админа (только `770511678`)

| Команда | Действие |
|---|---|
| `/menu` | меню кейсов: задать цену на каждый (кнопка → пришли число) |
| `/cancel` | отменить ввод цены |
| `/buyoff` | сбросить все цены (бот снова только уведомляет) |
| `/buystatus` | показать цены по кейсам |

Состояние автозакупки также видно админу в `/status`.

### Порядок безопасной обкатки

1. **DRY-RUN.** В env: `AUTOBUY_ENABLED=1` и `AUTOBUY_ITEMS=Revolution Case`
   (остальное по умолчанию → симуляция). После старта бот пришлёт «⏳ цены сброшены».
   Отправь `/menu` → нажми `Revolution Case` → пришли `0.31`. На дропе бот напишет
   тебе `🛒 🧪 dry-run: купил бы Revolution Case за X` и НИЧЕГО не потратит.
2. **Проверь поле цены.** Когда покупал вручную — какая цена реально списалась?
   Сравни с `price` и `storePrice` того предмета. Если списывается «store» —
   поставь `AUTOBUY_PRICE_FIELD=storePrice`. Иначе покупка может отвалиться по
   несовпадению цены.
3. **Боевой режим.** Задай `PIRATE_BEARER`, `PIRATE_STEAMID` и `AUTOBUY_DRY_RUN=0`.
   После рестарта снова открой `/menu` и выставь цены (они сбросились) — и бот
   начнёт реально покупать, присылая `🛒 ✅ Куплено: Revolution Case за X`.

### Что важно знать

- **После каждого рестарта цены сброшены.** Это нарочно: Render часто
  перезапускается, и без ручного `/menu` бот не потратит ничего. При старте
  админу приходит напоминание выставить цены.
- **Токен `PIRATE_BEARER` протухает.** Когда истечёт — покупка вернёт `401`, бот
  пришлёт админу `🔑 Токен автопокупки протух`. Поймай новый Bearer из запроса
  `start/v2` (F12 → Network) и обнови переменную на Render.
- **Kill switch.** `/buyoff` или `AUTOBUY_ENABLED=0` → покупки выключены,
  мониторинг и алерты продолжают работать.
- **Предохранители:** цена на каждый кейс через `/menu`, whitelist кейсов (`AUTOBUY_ITEMS`),
  дневной лимит, анти-ложная проверка hash, покупка только самого дешёвого нового.
- **Закупки видит только админ** (см. выше). Токен — только в env Render, никогда
  в код и не в чат.
